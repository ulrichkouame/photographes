## User Rights Implementation

### Right to Access (Art. 15)
- Users can download all their personal data from **Profile → Data Export**.
- The export is generated by the `export-user-data` Edge Function, which queries
  all tables containing the user's data and returns a JSON archive.
- Target delivery: **within 72 hours** of verified request.

### Right to Erasure (Art. 17)
- Users can delete their account from **Profile → Delete Account**.
- The `delete-user-data` Edge Function:
  1. Anonymises booking records (replaces PII with `[deleted]`).
  2. Deletes profile, messages, and media files from R2.
  3. Calls `supabase.auth.admin.deleteUser()` to remove the auth record.
- Financial/legal records are retained per the accounting obligation (10 years)
  in anonymised form.

### Right to Rectification (Art. 16)
- Users can update all profile fields from **Profile → Edit**.

### Right to Object / Restrict Processing (Art. 18, 21)
- Marketing consent can be withdrawn in **Profile → Notifications**.
- Analytics opt-out available via the cookie banner.

### Right to Data Portability (Art. 20)
- Included in the Data Export feature above (JSON format).

## Data Retention & Purge Schedule

Automated purge jobs run via Supabase `pg_cron`:

```sql
-- Delete expired marketing consents
SELECT cron.schedule(
  'purge-expired-consents',
  '0 3 * * *',  -- daily at 03:00 UTC
  $$DELETE FROM marketing_consents WHERE withdrawn_at < NOW() - INTERVAL '30 days'$$
);

-- Anonymise old deleted accounts
SELECT cron.schedule(
  'anonymise-deleted-accounts',
  '0 4 * * 0',  -- weekly on Sunday
  $$
  UPDATE profiles
  SET full_name = '[deleted]', phone = NULL, avatar_url = NULL
  WHERE deleted_at < NOW() - INTERVAL '30 days'
    AND full_name != '[deleted]'
  $$
);
```

## Data Breach Response Plan

1. **Detection** — Sentry / Datadog alerts or user report.
2. **Containment** — Revoke compromised tokens, restrict affected endpoints.
3. **Assessment** — Determine scope and categories of data affected.
4. **Notification**:
   - Supervisory authority (ARTCI in Côte d'Ivoire) within **72 hours**.
   - Affected users if high risk to their rights.
5. **Remediation** — Patch vulnerability, rotate secrets, post-mortem.
6. **Documentation** — Record in the internal data breach register.

## Privacy by Design Checklist

For each new feature, verify:

- [ ] Minimum data collection (only what is strictly necessary).
- [ ] New personal data fields added to the inventory table above.
- [ ] Retention period defined and automated purge configured.
- [ ] RLS policy restricts access to the data owner.
- [ ] Privacy impact assessment (PIA) completed for high-risk processing.
- [ ] Third-party receiving data is in the sub-processor list.
- [ ] Cookie banner updated if new tracking cookies are added.
